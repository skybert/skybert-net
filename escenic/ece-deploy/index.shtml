The ece-deploy command makes it easy to deploy a new EAR and, optionally, a DEB or RPM configuration package (in which case you'd also pass a --conf parameter):

  --ear  http://my.builder.com/snow/releases/snow-trunk-rev5233-2012-10-04_1608.ear \
  --update-publication-resources

There you go, deployment and rollback has never been easier. And this works regardless of this being a development VMware image, test, staging or production server. To get the ece-deploy command, you can either head over to the ece-scripts module on Github or you can install the escenic-content-engine-scripts package using the APT repository at apt.vizrt.com
