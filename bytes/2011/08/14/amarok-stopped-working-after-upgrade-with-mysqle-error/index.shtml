Amarok stopped working after upgrade with mysqle error

After upgrading to 2.4.1-1~bpo60+1 on my Debian box with Squeeze backports, Amarok stopped working, giving me an error dialogue with the message: GREPME MySQLe query failed! (2000) on init.

Starting amarok from the command line gave me a clue of what was wrong: It tried to read /etc/mysql/conf.d/, which on my machine didn't exist. I simply created it: 

# mkdir -p /etc/mysql/conf.d

After restarting Amarok, it started working again. With my setup of Amarok, nothing is written to this directory (or read for that matter, it's completely empty), but Amarok fails to start if it's not there. Not good. Luckily, it was an easy fix, though :-)
