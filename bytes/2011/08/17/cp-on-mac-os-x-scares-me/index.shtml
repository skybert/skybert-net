cp on Mac OS X Scares Me

I find the cp command on Mac to be quite scary. I wonder if this is due to its FreeBSD heritage. Mhmm...

I've got a directory a with two files: 

$ ls a/
total 0
-rw-r--r--  1 torstein  wheel     0B Aug 17 10:53 file1
-rw-r--r--  1 torstein  wheel     0B Aug 17 10:53 file2

And I have two empty directories, b and c. 

$ ls b/ c/
b/:

c/:

Now, I want to copy the entire a directory into my b directory, so I do: 

$ cp -r a b

Likewise, I want to copy it to the c directory too: 

$ cp -r a/ c

Now, I'd expect to see the same contents of the b and c directories: 

$ ls b
total 0
drwxr-xr-x  4 torstein  wheel   136B Aug 17 10:53 a
$ ls c
total 0
-rw-r--r--  1 torstein  wheel     0B Aug 17 10:53 file1
-rw-r--r--  1 torstein  wheel     0B Aug 17 10:53 file2

Whoa! What is this?!!. I know, I did add a slash to the second copy command but that can't surely make a difference, can it?

It turns out, it can. On GNU/Linux systems, you'll get the source directory included even if you add the slash. On GNU/Linux you must add a wildcard to say you just want the contained files to be copied: 

$ cp -r a/* b

However, adding just the slash doesn't alter behaviour of the copy command. However, it's quite different on Mac OS X. Scary stuff.
