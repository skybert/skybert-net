digging the rhino javascript cli

I've just started playing around with the Mozilla Rhino CLI tool, normally acessible with the command rhino or js.

The script lets you both run it in batch mode, getting validation feedback on your JS files, or interactively giving you a shell in which you can do fast JavaScript prototyping.

My only complaints so far is that the interactive shell doesn't have Emacs shortcuts (like the BASH, MySQL and PostgreSQL shells have) and that it doesn't follow the POSIX conventions for command line arguments. To remedy the first one, I can wrap it with rlwrap, I suppose ...
